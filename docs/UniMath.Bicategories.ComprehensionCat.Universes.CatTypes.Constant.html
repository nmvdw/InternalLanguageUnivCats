Codes for types in the empty context (for categories)
Our goal is to express that some universe in a category is closed under type formers, and in this file we consider the following: unit types, empty types, natural numbers, and subobject classifiers. Specifically, we define the following:
  • when a universe contains codes for these types
  • when a functor preserves those codes
We also develop the main infrastructure necessary to construct univalent displayed bicategories representing that the universe is closed under the aforementioned type formers.
There are several ideas in the development. The first important observation is that the aforementioned types (unit, empty, natural numbers, subobject classifier) are essentially determined by a type in the empty context. From the point of view of type theory, we can see it as follows: we have a type `ùüô` (unit type) in the empty context, and the unit type in any other context is obtained by weakening. For this reason, it suffices to add a code for the desired type only in the empty context meaning that we do not need to add coherence conditions.
To express that a universe contains some type, we use another idea. Specifically, we say that there is a code (i.e., morphism `ùüô --> u` from the terminal object to the universe type) whose associated type is isomorphic to the given type. This means that we assume that the category has enough structure (e.g., we assume that the category has an initial objects to express that the universe has codes for the empty type).
Finally, we give a uniform treatment of all these type formers. Specifically, we give a general definition that says when a universe contains a code for a given type. Note that this treatment is only for a fixed type in the empty context, because for other type formers (like ‚àè-types) one would need coherence conditions.

1. Codes for a type

  Definition type_in_cat_univ
             (A : C)
    : UU
    := ‚àë (a : ùüô --> univ_cat_universe C),
       z_iso (cat_el_map_el el a) A.

  Proposition isaset_type_in_cat_univ
              (A : C)
    : isaset (type_in_cat_univ A).
  Show proof.
    use isaset_total2.
    - apply homset_property.
    - intro.
      apply isaset_z_iso.

  Definition make_type_in_cat_univ
             {A : C}
             (a : ùüô --> univ_cat_universe C)
             (f : z_iso (cat_el_map_el el a) A)
    : type_in_cat_univ A
    := a ,, f.

  Definition type_in_cat_univ_code
             {A : C}
             (a : type_in_cat_univ A)
    : ùüô --> univ_cat_universe C
    := pr1 a.

  Definition type_in_cat_univ_z_iso
             {A : C}
             (a : type_in_cat_univ A)
    : z_iso (cat_el_map_el el (type_in_cat_univ_code a)) A
    := pr2 a.

  Proposition type_in_cat_univ_eq
              {A : C}
              {a‚ÇÅ a‚ÇÇ : type_in_cat_univ A}
              (p : type_in_cat_univ_code a‚ÇÅ = type_in_cat_univ_code a‚ÇÇ)
              (q : cat_el_map_el_eq el p ¬∑ type_in_cat_univ_z_iso a‚ÇÇ
                   =
                   type_in_cat_univ_z_iso a‚ÇÅ)
    : a‚ÇÅ = a‚ÇÇ.
  Show proof.
    induction a‚ÇÅ as [ a‚ÇÅ f‚ÇÅ ].
    induction a‚ÇÇ as [ a‚ÇÇ f‚ÇÇ ].
    cbn in *.
    induction p.
    apply maponpaths.
    use z_iso_eq.
    refine (!q @ _) ; cbn.
    apply id_left.

3. Preservation by functors

  Definition functor_preserves_type_in_cat_univ_code
             {A‚ÇÅ : C‚ÇÅ}
             {A‚ÇÇ : C‚ÇÇ}
             (FA : z_iso (F A‚ÇÅ) A‚ÇÇ)

             (a‚ÇÇ : type_in_cat_univ A‚ÇÇ)

       TerminalArrow _ _ ¬∑ type_in_cat_univ_code a‚ÇÇ.

  Definition functor_preserves_type_in_cat_univ_code_z_iso
             {A‚ÇÅ : C‚ÇÅ}
             {A‚ÇÇ : C‚ÇÇ}
             {FA : z_iso (F A‚ÇÅ) A‚ÇÇ}
             {a‚ÇÅ : type_in_cat_univ A‚ÇÅ}
             {a‚ÇÇ : type_in_cat_univ A‚ÇÇ}

       ¬∑ cat_el_map_pb_mor _ _ _

             {A‚ÇÅ : C‚ÇÅ}

    - intros.
      apply homset_property.
